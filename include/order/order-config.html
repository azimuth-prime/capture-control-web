Sales Order Configuration

Configure workflows and settings for Sales Orders.

Sales Order Stock
Configure how stock is handled for Sales Orders
Indicate whether orders can contain back ordered items. If they cannot then items with 0 inventory will not be available to add to orders.
Indicate whether orders can be split shipped, ie: ship some items if available, or an order must be complete before shipping.
Indicate whether backordered items should be fulfilled immediately upon receipt or wait until the entire backordered amount on the order is available.
By default the system will allocate inventory based on FIFO (First In First Out) however you can select alternative methods below such as by expiry date as well as by location counts.
Sales Order Warehouse Fulfillment
Configure how warehouse fulfillment is handled.
Indicate whether sales orders should be fulfilled by multiple physical warehouses. Only applies if more than one physical warehouse is defined within the system. When disabled the warehouse selected during the order process will be used for the entire sales order.
When multiple warehouses are used the system will select the closest warehouse to the sales order shipping address first then move outward to the next closest to fulfill the order.
When multiple warehouses are used the system will only use physical warehouses within the same national borders of the sales order shipping address country.
Invoicing
Configure how invoicing is handled by the system.
Indicate whether Invoices should be created for every Sales Order. If manually is selected an invoice will only be created by explicitly clicking on the create invoice button on the Sales Order details page.
Indicate whether invoices should be automatically emailed to the billing address email once orders are shipped. If manually is selected then invoices can be mailed by clicking the email invoice button in the Sales Order details page.
Drop Ship
Configure how drop ship items and sales orders are handled by the system. Due to the nature of drop shipping the system processes are automated based on the selections below.
Indicate whether the system will support drop shipping. Note: Drop Ship availability is based on the SKU. If drop shipping id disabled the SKUs denoted as drop ship will not be available for sale.
Indicate whether dropship suppliers should send direct to the customer or first send to your warehouse for final shipping. If this is enabled the supplier will also receive a packing slip along with the Purchase Order for the dropship sales order. If disabled then dropship items will be backordered in the sales order until received.
